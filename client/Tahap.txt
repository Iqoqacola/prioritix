Tahap 2: Struktur Folder & Tipe Data (The Blueprint)
Sebelum coding UI, kita harus tentukan dulu struktur datanya. Ini agar kita tidak bingung "data todo isinya apa aja ya?".
Buat Struktur Folder di dalam src/:
components/ (Tombol, Modal, Card)
pages/ (Dashboard, Login)
services/ (Mock API/Logic data)
hooks/ (Custom hooks jika perlu)
Definisikan Tipe Data (src/types.ts)
Buat file src/types.ts. Kita butuh tipe untuk User, Todo, dan Status.
code
TypeScript
export enum Priority {
  LOW = 'low',
  MEDIUM = 'medium',
  HIGH = 'high',
  URGENT = 'urgent',
}

export enum Status {
  TODO = 'todo',
  IN_PROGRESS = 'in_progress',
  DONE = 'done',
}

export interface Todo {
  id: string;
  title: string;
  description?: string;
  status: Status;
  priority: Priority;
  dueDate?: string;
  tags?: string[]; // Fitur tags yang diminta
  createdAt: string;
}

export interface User {
  id: string;
  name: string;
  email: string;
  avatarUrl?: string;
}

Tahap 3: Mock Service (Backend-backend-an)
Karena kita belum pakai Database beneran, kita buat simulasi "Service" yang menyimpan data di localStorage browser. Ini membuat app tetap jalan meski di-refresh.
Buat src/services/mockService.ts
Di sini kita buat fungsi getTodos, createTodo, dll.
Konsep Kuncinya:
Ambil string dari LocalStorage.
Parse jadi JSON.
Manipulasi Array (push, filter, map).
Simpan balik ke LocalStorage.
(Kamu bisa lihat referensi kode mockService.ts dari jawaban saya sebelumnya, tapi cobalah tulis ulang fungsi getTodos dan createTodo saja dulu untuk latihan).

Tahap 5: Halaman & Routing (Navigasi)
Sekarang kita atur alur aplikasinya.
Buat File Halaman (Kosongan dulu)
src/components/LandingPage.tsx (Halaman depan)
src/components/Auth.tsx (Halaman Login)
src/components/Dashboard.tsx (Inti aplikasi)
Setup Router di src/App.tsx
Kita gunakan react-router-dom.
code
Tsx
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';
// Import halaman-halaman tadi

function App() {
  // Nanti di sini kita cek apakah user sudah login atau belum
  const user = null; // Sementara null dulu

  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<LandingPage />} />
        <Route path="/auth" element={<Auth />} />
        <Route path="/dashboard" element={user ? <Dashboard /> : <Navigate to="/auth" />} />
      </Routes>
    </BrowserRouter>
  );
}

Tahap 7: Fitur Lanjutan (Search, Filter, Kanban)
Setelah list dasar jadi, baru kita tambahkan "bumbu" fitur menarik tadi.
Search & Filter
Buat state searchQuery.
Hubungkan ke input text di Header.
Sebelum melakukan .map untuk render, lakukan .filter() dulu pada array todos.
code
JavaScript
const filteredTodos = todos.filter(todo => 
   todo.title.toLowerCase().includes(searchQuery.toLowerCase())
);
Pomodoro Timer
Buat komponen PomodoroTimer.tsx. Gunakan useEffect dengan setInterval yang berjalan setiap 1 detik (1000ms) untuk mengurangi waktu hitung mundur.
Kanban View
Buat state viewMode ('list' atau 'board').
Jika 'board', render 3 kolom (To Do, In Progress, Done).
Di setiap kolom, filter lagi todos yang statusnya sesuai kolom tersebut.
Tips Belajar:
Jangan hafal kodenya. Pahami alurnya: Data diambil -> Disimpan di State -> State berubah -> UI otomatis berubah.
Eksperimen Tailwind. Coba-coba ganti bg-slate-50 jadi warna lain, atau mainkan p-4 vs p-8 untuk melihat bedanya.
